#前端缓存 - 分为http缓存和浏览器缓存

##http缓存
```bash
http缓存分为强缓存 和 协商缓存

http缓存都是从第二次请求开始的， 第一次请求资源时 服务器返回资源 并在response header
头中回传资源的缓存参数 第二次 请求时 浏览器会判断这些请求参数是否击中强缓存 
如果击中强缓存 返回200 否则就会把请求参数放入到response header中传回给服务器 查看是否击中
返回协商缓存 击中返回304 否则返回新资源

浏览器缓存 cookie，ls，ss
cookie 用来存储用户信息 4kb 20条, 会在请求时传递
ls, ss 5MB 

```
